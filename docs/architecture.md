# Архитектура системы

Приложение реализовано на основе микросервисного архитектурного подхода.
Каждая бизнес-область выделена в отдельный логический сервис.
Взаимодействие между клиентской частью и сервисами осуществляется
через API Gateway.

## Выделенные сервисы

### Auth Service
Отвечает за:
- регистрацию пользователей
- аутентификацию
- выдачу JWT-токенов
- хранение ролей пользователей

### User Service
Отвечает за:
- управление профилем пользователя
- навыки, интересы
- рейтинг и отзывы
- портфолио волонтёра

### Project Service
Отвечает за:
- создание и управление мероприятиями
- статусы мероприятий
- загрузку документов

### Application Service
Отвечает за:
- обработку заявок на участие
- статусы заявок
- связь волонтёров и мероприятий

### Chat Service
Отвечает за:
- обмен сообщениями
- групповые чаты мероприятий
- хранение истории сообщений

### Notification Service
Отвечает за:
- email-уведомления
- системные уведомления

### Admin Service
Отвечает за:
- управление пользователями
- модерацию мероприятий и отзывов
- аналитику и статистику системы
